QSAR Model Reporting Format (QMRF), version 1.2

Background

The set of information that you provide will be used to facilitate regulatory considerations of (Q)SARs. For this purpose, the structure of the QMRF is devised to reflect as much as possible the OECD principles for the validation, for regulatory purposes, of (Q)SAR models. Please, try to fill in the fields of the QMRF for the model of interest. If the field is not pertinent with the model you are describing, or if you cannot provide the requested information, please answer “no information available”. You are invited to consult the OECD "Guidance document on the Validation of (Quantitative) Structure-Activity Relationship Models" that can aid you in filling in a number of fields of the QMRF.

Instructions

  • Search QMRF inventory by
    • Free text search
    • Structure and similarity search
  • Download QMRF documents as
    • native XML format (.xml)
    • Portable Document Format (.pdf)
    • Microsoft Excel worksheets (.xls)
    • Attachment (chemical structures in .sdf and other information in the original format)
  • Submit new QMRF documents
    • QMRF documents can be edited and submitted by registered users only. Use Register link to register and log into the Inventory by Log in link
    • QMRF documents can be saved as Draft or submitted to the Inventory.
    • Once submitted, it will be reviewed by administrator for any missing or unclear information.
    • The reviewer might publish the document, or return it to the user for further editing.
    • Only published documents are visible for all users and searchable.
  • Create QMRF documents offline
    • Create QMRF by QMRF Editor. Once started, it will create shortcut on your desktop and can be started later even offline.
    • Save as QMRF file locally
    • Log in as a registered user, select New document and load your file into the QMRF applet.

Publish

Click publish button will assign QMRF number and publish the QMRF document.
QMRF number is an unique number (numeric identifier) is assigned to any QMRF that is published in the ECB inventory. The number encodes the following information: model described in the QMRF (as derived from field 4.2), software implementing the model (as derived from field 1.3), version of the QMRF for the same model and the same software (as derived from the information included in field 2.4) and author of the QMRF (as derived from field 2.2). The number is unique for any QMRF uploaded and stored in the ECB inventory.
Click "Retrieve number from catalog" to obtain corresponding numbers.
If one of the fields is missing, return the document to the author to insert the missing data.
